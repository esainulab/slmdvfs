SECTION 3.5: DVFS CONTROL METHODOLOGY
================================================================================

GPU FREQUENCY CONTROL:
  Tool:      Linux devfreq sysfs interface (NOT jetson_clocks, NOT nvpmodel)
  Script:    run_gpu_freq_experiment_qnli.sh (and per-model variants)
  Procedure:
    1. Set governor to "userspace":
       sudo bash -c "echo userspace > /sys/class/devfreq/17000000.gpu/governor"
    2. Pin frequency (both max and min to same value):
       sudo bash -c "echo <freq_hz> > /sys/class/devfreq/17000000.gpu/max_freq"
       sudo bash -c "echo <freq_hz> > /sys/class/devfreq/17000000.gpu/min_freq"
    3. Wait 2 seconds for stabilization
    4. Verify: cat /sys/class/devfreq/17000000.gpu/cur_freq
  Baseline:  Restore "nvhost_podgov" dynamic governor (uncapped)

FREQUENCY POINTS:
  Source:    All 11 hardware-supported discrete levels from available_frequencies
  Rationale: Not user-selected -- these are the only frequencies the hardware supports
  Spacing:   ~102 MHz per step (hardware-defined, approximately uniform)

CPU FREQUENCY:   NOT controlled (system cpufreq governor, auto)
EMC FREQUENCY:   NOT controlled (EMC governor, auto, ~2100-3200 MHz observed)

BASELINE CONFIGURATIONS:
  Mode 0 (MAXN):
    GPU:          Dynamic governor (nvhost_podgov), observed ~1300 MHz during training
    Power:        No cap (unconstrained)
    Purpose:      Maximum performance reference

  Mode 2 (MODE_30W):
    GPU:          Dynamic governor, constrained by 30W system budget

SECTION 3.6: POWER AND ENERGY MEASUREMENT
================================================================================

TOOL: tegrastats (NVIDIA Jetson system utility)
  Sampling rate:    10 ms intervals (--interval 10)
  Output:           Raw text log -> parsed to CSV by parse_tegrastats_labeled.py

POWER CHANNELS MEASURED:
  VDD_GPU_SOC:      GPU + SoC subsystem power (mW)   <- used in energy calc
  VDD_CPU_CV:       CPU + CV engine power (mW)        <- used in energy calc
  VIN_SYS_5V0:      Total 5V system input power (mW)  <- logged but NOT in energy calc
  Note: Energy uses CPU+GPU channels only, not the full-system VIN_SYS_5V0 rail

OTHER METRICS LOGGED (42 CSV columns total):
  RAM used/total (MB), swap, CPU load per core (%), CPU freq per core (MHz),
  EMC utilization (%), EMC freq (MHz), GPU utilization (%), GPU freq (MHz),
  hardware engine utilizations (nvenc, nvdec, nvjpg, vic, ofa, nvdla, pva),
  per-zone temperatures (cpu, soc0, soc1, soc2, tj in degC)

ENERGY CALCULATION:
  Formula:    E (J) = mean(VDD_GPU_SOC + VDD_CPU_CV) [mW] x train_time [s] / 1000
  Method:     Average power over training window multiplied by wall-clock duration
              NOTE: Simple mean x time, NOT trapezoidal integration
  Code:       parse_tegrastats_labeled.py -> compute_derived_metrics()
              total_power_mw = power_VDD_GPU_SOC_mw + power_VDD_CPU_CV_mw (per sample)
              energy_j = (total_power_mw * train_time) / 1000.0

TRAINING TIME:
  Method:     Wall-clock time extracted from Python output log
              Regex: "Train wall time: <X> s"
  Units:      Seconds

ACCURACY:
  Metric:     Classification accuracy (%) on GLUE validation split
  Evaluation: Single pass after training completes (eval_every_epoch=False)
  Note:       GLUE test labels not publicly available; validation set used

SECTION 3.7: EXPERIMENTAL PROCEDURE
================================================================================

PROTOCOL (per model x dataset x frequency):
  1.  Set GPU governor to "userspace"
  2.  Pin GPU frequency (max_freq = min_freq = target)
  3.  Wait 2s, verify cur_freq
  4.  Check temperature (abort if > 85 degC, wait if > 75 degC)
  5.  Launch tegrastats --interval 10 --logfile <outdir>/tegrastats_measure.txt
  6.  Run: python BERT_sst2_FullFT.py --model_name ... --dataset ... [args]
  7.  Kill tegrastats after training exits
  8.  Run parse_tegrastats_labeled.py -> CSV + Google Sheets upload
  9.  Sleep 30-60s cooldown

REPETITIONS:
  Each configuration run ONCE (no repeated trials)
